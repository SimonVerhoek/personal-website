Appollo

FUNCTIONAL DESIGN
(FEBRUARY - MARCH 2014)

For one of my University's courses, App Design, our task was to design the architecture for a mobile application from the ground up.

Amongst the enormous amount of new music being released these days, finding that unforgettable new song can be quite daunting. Contextual Inquiry research concluded that listening to music is not only personal and heavily context-dependent, but also a relaxing experience that should take as little effort as possible. Our goal thus became to design a service that would make discovering new nice music effortless and tailormade.

Our result was Appollo: a customizable music streaming service that automatically adapts to the feedback you give through a simple like/dislike system. The music stream you receive comes from a library filled with newly released music of many different sorts, extensively tagged with metadata by the artist (BPM, (sub)genre, paired emotions etc.). While (dis)liking a song, you score the tags attacted to that song. A powerful algorithm will then take the tags you prefer the most, and use those to carefully select the next songs to stream.

DormBoard

WEB APPLICATION
languages used: PHP, SQL, Javascript, HTML, CSS, MVC, Bootstrap
(May 2014 - now)

DormBoard is an organisation tool for students. It allows the members of student dormitories to keep track of a cooking and dining schedule, shopping list and the according finances on a shared online board.

I have built this application from scratch, both front end and back end, based on the MVC pattern. The back end consists of a mySQL database, driven by PHP code according to this moment's security standards. The front end is built upon the popular Twitter Bootstrap framework, paired with custom HTML, CSS, JavaScript and jQuery.

For testing purposes, I am currently deploying this app via git to Openshift.

As of today, DormBoard isn't fully finished yet. My goal is to have it ready for a first release by the end of 2014.
